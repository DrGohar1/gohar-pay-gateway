# المستخدمون التجريبيون — Gohar Pay Gateway

> **مهم**: المشروع لا يحتوي على مستخدمين مُسبقي الإنشاء. عند أول `signup` بأي إيميل، يتم تشغيل trigger `handle_new_user` الذي يستدعي `bootstrap_demo_merchant` لإنشاء **تاجر تجريبي كامل** مع كل البيانات (50 حوالة، 14 طلب، 3 مصادر دفع، 4 تنبيهات، إلخ).

## كيف تجرب النظام

### 1. مستخدم تاجر (Merchant Owner)
- افتح `/signup`
- استخدم أي إيميل (مثل `merchant@test.com`) وكلمة مرور 8 أحرف+
- بعد الدخول تذهب تلقائياً إلى `/app` ولوحة التاجر جاهزة بالبيانات
- يمكنك تصفح كل الـ 14 صفحة فوراً

### 2. مستخدم مسؤول داخلي (Internal Admin)
بعد إنشاء حسابك، شغّل في قاعدة البيانات (عبر Lovable Cloud → SQL):

```sql
INSERT INTO public.user_roles (user_id, role)
SELECT id, 'internal_admin'::app_role FROM auth.users WHERE email = 'admin@test.com';
```

ثم افتح `/admin` للوصول إلى:
- `/admin` — لوحة تحكم بالإحصائيات
- `/admin/merchants` — إدارة جميع التجار
- `/admin/devices` — أجهزة Android المتصلة
- `/admin/parser` — قوالب تحليل الرسائل
- `/admin/fraud` — قائمة المخاطر والاحتيال
- `/admin/system` — الباقات وإعدادات النظام

### 3. إعادة تعيين كلمة المرور
- من `/login` اضغط "نسيت كلمة المرور"
- ستصلك رسالة بريد مع رابط يوجهك إلى `/reset-password`

## ملاحظات أمنية

- لا يمكنك الترقية لـ `internal_admin` عبر الواجهة — فقط عبر SQL مباشرة (هذا متعمد لمنع تصعيد الصلاحيات)
- جميع البيانات معزولة بـ RLS — تاجر لا يرى بيانات تاجر آخر
- مفاتيح API تُخزن hash فقط (SHA-256)، لا يمكن استعادتها بعد الإنشاء
